# Phase 2 — Generation

**Goal:** `agent-policy init` and `agent-policy generate` work end-to-end. A valid `agent-policy.yaml` produces `AGENTS.md`, `CLAUDE.md`, and/or `.cursor/rules/default.mdc` on disk. Golden snapshot tests cover all renders. The CLI is fully wired.

**Depends on:** Phase 1 (normalized `Policy` model, load pipeline)  
**Unlocks:** Phase 3 (check command reuses the render pipeline in memory)

---

## Overview

The render pipeline takes a normalized `Policy` and produces one or more `RenderedOutput` values — each containing a relative file path and a content string. The `generate` command then writes those to disk.

```
Policy (from Phase 1)
    ↓ render_all()
Vec<RenderedOutput>  { path, content }
    ↓ write_atomic()
files on disk
```

Keeping the render pipeline separate from file I/O is critical for Phase 3: the `check` command uses `render_all()` without touching the disk.

## Rendered Output Type (`src/render/mod.rs`)

```rust
use camino::Utf8PathBuf;
use crate::{error::Result, model::normalized::Policy};

pub mod agents_md;
pub mod claude_md;
pub mod cursor_rules;

/// A single rendered output file.
pub struct RenderedOutput {
    /// Relative path from the repo root where this file should be written.
    pub path: Utf8PathBuf,
    /// The rendered string content.
    pub content: String,
}

/// Render all outputs enabled by the policy.
///
/// Returns a list of outputs in a deterministic order:
/// AGENTS.md → CLAUDE.md → cursor rules.
pub fn render_all(policy: &Policy) -> Result<Vec<RenderedOutput>> {
    let mut outputs = Vec::new();
    if policy.outputs.agents_md {
        outputs.push(agents_md::render(policy)?);
    }
    if policy.outputs.claude_md {
        outputs.push(claude_md::render(policy)?);
    }
    if policy.outputs.cursor_rules {
        outputs.push(cursor_rules::render(policy)?);
    }
    Ok(outputs)
}
```

## Template Design

Templates live in `templates/` and are embedded at compile time via `include_str!`. This means the binary has no runtime file dependencies — consuming repos do not need to install template files.

### Why `include_str!` over `rust-embed` or external file loading

- Zero runtime dependencies
- No install-step for templates
- Template content is versioned with the binary release
- Compile-time error if a template file is missing

## Renderer Implementation

Each renderer embeds its template and creates a `minijinja::Environment` scoped to that render call.

### `src/render/agents_md.rs`

```rust
use crate::{error::{Error, Result}, model::normalized::Policy, render::RenderedOutput};
use camino::Utf8PathBuf;
use minijinja::Environment;

const TEMPLATE: &str = include_str!("../../templates/AGENTS.md.j2");

pub fn render(policy: &Policy) -> Result<RenderedOutput> {
    let mut env = Environment::new();
    env.add_template("AGENTS.md", TEMPLATE)
        .map_err(|e| Error::Render { target: "AGENTS.md".to_owned(), source: e })?;

    let tmpl = env.get_template("AGENTS.md")
        .map_err(|e| Error::Render { target: "AGENTS.md".to_owned(), source: e })?;

    let content = tmpl
        .render(minijinja::context! {
            project => &policy.project,
            commands => &policy.commands,
            // `commands_defined` is a precomputed bool so templates can check whether any
            // command is set without calling a Rust method from Jinja2 context.
            commands_defined => !policy.commands.is_empty(),
            paths => &policy.paths,
            roles => &policy.roles,
            constraints => &policy.constraints,
        })
        .map_err(|e| Error::Render { target: "AGENTS.md".to_owned(), source: e })?;

    Ok(RenderedOutput {
        path: Utf8PathBuf::from("AGENTS.md"),
        content,
    })
}
```

> `claude_md.rs` and `cursor_rules.rs` follow the same pattern, with their respective `TEMPLATE` const, target name string, and output path (`CLAUDE.md` and `.cursor/rules/default.mdc`). **Both must also pass `commands_defined` and use references in the `context!` call.**

**Important:** For `minijinja` to serialize the model types, all `Policy` sub-types need to implement `serde::Serialize`. Add `#[derive(Serialize)]` to every type in `src/model/normalized.rs` and `src/model/targets.rs`.

Two additional rules for all renderers:

1. **Always pass references** (`&policy.commands` not `policy.commands`) — `policy` is a `&Policy` borrow and moving fields out of it won't compile. Since `T: Serialize` implies `&T: Serialize`, references work correctly with `context!`.
2. **Precompute booleans for Rust methods** — Jinja2 context variables are serialized field values, not any Rust method. Any `impl` method (like `commands.is_empty()`) must be evaluated in Rust before the `context!` call and passed as a named boolean. Template should use `{% if not commands_defined %}` not `{% if commands.is_empty %}`.

## File Writing (`src/util/fs.rs`)

Atomic write: write to a `.tmp` file alongside the target, then rename. This is safe against partial writes.

```rust
use crate::error::{Error, Result};
use std::path::Path;
use std::{fs, io};

/// Write `content` to `path` atomically.
///
/// Creates parent directories if they do not exist.
/// Writes to a temporary file alongside `path`, then renames atomically.
pub fn write_atomic(path: &Path, content: &str) -> Result<()> {
    if let Some(parent) = path.parent() {
        fs::create_dir_all(parent).map_err(|e| Error::Io {
            path: parent.to_owned(),
            source: e,
        })?;
    }

    let tmp = path.with_extension("agent-policy.tmp");
    fs::write(&tmp, content).map_err(|e| Error::Io {
        path: tmp.clone(),
        source: e,
    })?;

    fs::rename(&tmp, path).map_err(|e| Error::Io {
        path: path.to_owned(),
        source: e,
    })?;

    Ok(())
}

/// Read a file to string, returning None if the file does not exist.
pub fn read_if_exists(path: &Path) -> Result<Option<String>> {
    match fs::read_to_string(path) {
        Ok(s) => Ok(Some(s)),
        Err(e) if e.kind() == io::ErrorKind::NotFound => Ok(None),
        Err(e) => Err(Error::Io { path: path.to_owned(), source: e }),
    }
}
```

```rust
use crate::error::{Error, Result};
use camino::Utf8Path;
use std::fs;

const STARTER_YAML: &str = r#"# agent-policy.yaml
# Generated by: agent-policy init
# Schema: https://github.com/CameronBrooks11/agent-policy/agent-policy.schema.json
#
# Edit this file, then run: agent-policy generate

project:
  name: my-project
  summary: A short description of this repository.

# Shell commands for agents to use in this repo.
# Uncomment and fill in any that apply to this project.
# commands:
#   install: "npm install"
#   dev: "npm run dev"
#   lint: "npm run lint"
#   test: "npm test"
#   build: "npm run build"

# Classify repository paths using glob patterns
paths:
  editable:
    - src/**
    - docs/**
  protected:
    - .github/workflows/**
    - deployment.json
  generated:
    - AGENTS.md
    - CLAUDE.md
    - .cursor/rules/**

# Define named agent roles with scoped permissions (optional)
# roles:
#   docs_agent:
#     editable:
#       - docs/**
#     forbidden:
#       - src/**

# Behavioral guardrails (optional)
constraints:
  forbid_secrets: true
  require_tests_for_code_changes: false
  require_human_review_for_protected_paths: true

# Which output targets to generate.
# Valid: agents-md, claude-md, cursor-rules
# Omit this section to use the default (agents-md only).
outputs:
  - agents-md
"#;

pub fn run(force: bool) -> Result<()> {
    let path = Utf8Path::new("agent-policy.yaml");

    if path.exists() && !force {
        return Err(Error::Io {
            path: path.as_std_path().to_owned(),
            source: std::io::Error::new(
                std::io::ErrorKind::AlreadyExists,
                "agent-policy.yaml already exists. Use --force to overwrite.",
            ),
        });
    }

    fs::write(path, STARTER_YAML).map_err(|e| Error::Io {
        path: path.as_std_path().to_owned(),
        source: e,
    })?;

    println!("Created agent-policy.yaml");
    println!("Edit it, then run: agent-policy generate");
    Ok(())
}
```

### `src/commands/generate.rs`

```rust
use crate::{error::Result, load, model::normalize, render, util::fs::write_atomic};
use camino::Utf8Path;

pub fn run(config: &Utf8Path) -> Result<()> {
    let raw = load::load_file(config)?;
    let policy = normalize(raw)?;
    let outputs = render::render_all(&policy)?;

    for output in &outputs {
        write_atomic(output.path.as_std_path(), &output.content)?;
        println!("  wrote  {}", output.path);
    }

    println!("\nGenerated {} file(s).", outputs.len());
    Ok(())
}
```

```rust
use agent_policy::cli::{Cli, Command};

fn main() {

        eprintln!("error: {e}");
        process::exit(1);
    }
}
```

The binary exits with code `1` on any error. This is what CI expects.

## Golden Snapshot Tests (`tests/golden.rs`)

Golden tests use `insta` for snapshot management. The first run with a new snapshot creates the `.snap` file; subsequent runs compare against it.

```rust
use agent_policy::{load::load_str, model::normalize, render::render_all};

fn render_yaml(yaml: &str) -> Vec<(String, String)> {
    let raw = load_str(yaml).expect("parse failed");
    let policy = normalize(raw).expect("normalize failed");
    let outputs = render_all(&policy).expect("render failed");
    outputs.into_iter().map(|o| (o.path.to_string(), o.content)).collect()
}

#[test]
fn golden_minimal() {
    let yaml = std::fs::read_to_string("examples/minimal/agent-policy.yaml").unwrap();
    for (path, content) in render_yaml(&yaml) {
        // insta uses the test name + path as the snapshot name
        insta::assert_snapshot!(path, content);
    }
}

#[test]
fn golden_all_outputs_enabled() {
    let yaml = r#"
project:
  name: website
  summary: Marketing website.
commands:
  install: npm install
  test: npm test
  lint: npm run lint
paths:
  editable:
    - src/**
    - docs/**
  protected:
    - .github/**
roles:
  docs_agent:
    editable:
      - docs/**
    forbidden:
      - src/**
constraints:
  forbid_secrets: true
  require_tests_for_code_changes: true
outputs:
  - agents-md
  - claude-md
  - cursor-rules
"#;
    for (path, content) in render_yaml(yaml) {
        insta::assert_snapshot!(path, content);
    }
}

#[test]
fn golden_no_commands_no_roles() {
    let yaml = r#"
project:
  name: bare-bones
outputs:
  - agents-md
"#;
    for (path, content) in render_yaml(yaml) {
        insta::assert_snapshot!(path, content);
    }
}
```

**Snapshot workflow:**

1. Run `cargo test` — snapshots are created in `tests/snapshots/` with `.snap` extension.
2. Review generated snapshots: run `cargo insta review` to approve or reject.
3. Commit approved snapshots. They are fixtures from this point forward.
4. Any future change to templates will cause snapshot tests to fail — run `cargo insta review` to update.

## Exit Condition

Phase 2 is complete when all of the following are true:

- [ ] All three templates (`AGENTS.md.j2`, `CLAUDE.md.j2`, `cursor_rule.mdc.j2`) are written and produce valid, readable output
- [ ] All normalized model types derive `serde::Serialize` so `minijinja` can access them
- [ ] All renderers pass context values as references (`&policy.commands`) and pass a precomputed `commands_defined: bool` boolean
- [ ] `render_all()` is implemented and dispatches correctly based on `policy.outputs`
- [ ] `write_atomic()` is implemented in `util/fs.rs`
- [ ] `init` command writes a well-commented starter YAML (commands section fully commented out, not null-valued) and refuses to overwrite without `--force`
- [ ] `generate` command loads, validates, normalizes, renders, and writes all enabled outputs
- [ ] `generate` prints each file path written and a summary count
- [ ] `main.rs` is wired to all three commands; `Check` exits with a clear "not yet implemented" message (Phase 3 fills in the real implementation)
- [ ] `cargo build --release` produces a working binary
- [ ] Golden snapshot tests pass for at least: minimal, full config (all outputs), bare-bones (no optional fields)
- [ ] All CLI integration tests pass
- [ ] `cargo test` passes with zero failures
- [ ] `cargo clippy --all-targets -- -D warnings` passes
- [ ] `cargo doc --no-deps` passes with no warnings
- [ ] A human has reviewed the rendered AGENTS.md output and confirmed it reads well
